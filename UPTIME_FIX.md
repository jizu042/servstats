# Исправление проблемы с Uptime и Статистикой

## Проблема
- Uptime сбрасывался при перезагрузке страницы
- Новые пользователи видели uptime с нуля
- Статистика не сохранялась между сессиями

## Причина
Данные хранились в `localStorage` браузера, а не в базе данных.

## Решение

### Что изменено:

1. **Uptime теперь всегда из БД**
   - Удалено сохранение в `localStorage`
   - API всегда возвращает `onlineSince` из базы данных
   - Анализируется до 300 последних сэмплов для точности

2. **Статистика из БД**
   - Удалено локальное накопление статистики
   - Все данные загружаются из API
   - История игроков берётся из `player_sessions` таблицы

3. **Улучшенная логика на бэкенде**
   - Если сервер онлайн - ищется начало текущего онлайн-стрика
   - Если сервер оффлайн - `onlineSince = null`
   - Если нет данных в БД - используется текущее время

### Файлы изменены:
- `server/src/index.js` - улучшен расчёт `onlineSince`
- `web/src/hooks/useServerMonitor.js` - удалено использование localStorage
- `web/src/contexts/ServerContext.jsx` - удалено локальное накопление статистики

## Как это работает теперь:

1. **Worker** собирает данные каждые 5 минут и сохраняет в БД
2. **API** читает данные из БД и вычисляет `onlineSince`
3. **Frontend** отображает данные из API (не из localStorage)

## Результат:

✅ Uptime не сбрасывается при перезагрузке страницы
✅ Новые пользователи видят реальный uptime сервера
✅ Статистика одинакова для всех пользователей
✅ Данные сохраняются между сессиями

## Важно!

После деплоя нужно:
1. Исправить базу данных (см. `DATABASE_FIX.md`)
2. Подождать 5-10 минут, пока worker соберёт первые данные
3. Обновить страницу

Uptime начнёт отображаться корректно после того, как worker соберёт несколько сэмплов.

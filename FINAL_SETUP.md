## 📊 Как это работает

### Автоматический сбор данных:

1. **При каждом запросе /api/status** (когда пользователь открывает сайт):
   - Сохраняется sample в БД
   - Отслеживаются сессии игроков

2. **Каждые 5 минут** (встроенный collector):
   - Опрашивает все серверы из MONITOR_SERVERS
   - Сохраняет статистику
   - Открывает/закрывает сессии игроков
   - Очищает старые данные (30 дней)

### Источники данных (в порядке приоритета):

1. **Direct** - прямое подключение к серверу (лучшее качество)
2. **ismcserver.online** - если есть ISMCSERVER_TOKEN
3. **mcstatus.io** - бесплатный API
4. **mcsrvstat.us** - бесплатный API (новый!)

## 🔧 Дополнительные настройки (опционально)

Можете добавить в Environment:

```env
# Частота сбора данных (по умолчанию 5 минут)
POLL_INTERVAL_MS=300000

# Таймаут запросов (по умолчанию 8 секунд)
POLL_TIMEOUT_MS=8000

# Хранить данные N дней (по умолчанию 30)
STATS_RAW_RETENTION_DAYS=30

# Предпочитаемый источник: auto, direct, ismcserver, mcstatus, mcsrvstat
API_SOURCE_PREFERENCE=auto

# Токен для ismcserver.online (опционально)
ISMCSERVER_TOKEN=your_token_here
```

## 🐛 Если что-то не работает

### Ошибка "database 'db' does not exist"
**Причина**: DATABASE_URL не обновлён  
**Решение**: Выполните Шаг 1

### Collector не запускается
**Причина**: MONITOR_SERVERS не настроен  
**Решение**: Выполните Шаг 2

### Данные не собираются
**Причина**: Сервер недоступен или все API источники недоступны  
**Решение**: Проверьте логи API, убедитесь что сервер Minecraft доступен

### Uptime сбрасывается
**Причина**: Недостаточно данных в БД  
**Решение**: Подождите 10-15 минут после первого запуска
